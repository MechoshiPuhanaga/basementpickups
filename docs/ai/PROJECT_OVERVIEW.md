# Project Overview — Basement Pickups

Basement Pickups is a boutique guitar pickup brand focused on handcrafted tone, premium presentation, and a strong visual identity.

The website is both:

- an ecommerce experience
- and a brand presentation platform

The goal is not to feel like a generic ecommerce store.

The experience should feel:

- curated
- premium
- cinematic
- handcrafted
- restrained
- elegant
- atmospheric

The site should create the feeling of:

- boutique luthier craftsmanship
- luxury audio equipment
- vintage industrial elegance
- high-end workshop atmosphere
- musician-focused authenticity

---

# Brand Personality

The brand should feel:

- confident but not loud
- artistic but disciplined
- premium but not corporate
- dark but warm
- technical but emotional
- elegant but rugged

Avoid:

- aggressive marketing language
- exaggerated hype
- flashy ecommerce patterns
- generic startup aesthetics
- excessive UI decoration

The tone should feel:

- mature
- deliberate
- trustworthy
- crafted
- timeless

---

# Target Audience

The audience includes:

- guitar players
- tone enthusiasts
- boutique gear buyers
- musicians who value craftsmanship
- players who appreciate premium presentation
- customers familiar with vintage and boutique guitar culture

The audience is likely visually sensitive and design-aware.

The experience should reward attention to detail.

---

# Product Philosophy

Products are handcrafted guitar pickups.

The presentation should emphasize:

- craftsmanship
- materials
- tone character
- individuality
- premium construction
- careful engineering

Products should feel like:

- instruments
- crafted objects
- precision components

not mass-produced commodity products.

---

# Visual Philosophy

The visual system is heavily inspired by:

- Art Deco geometry
- vintage cinema
- luxury product photography
- boutique workshop aesthetics
- elegant industrial design
- dark editorial layouts

The visual identity depends heavily on:

- dark backgrounds
- gold accents
- controlled contrast
- geometric framing
- restrained typography
- cinematic imagery
- spacious composition

The visual system should feel cohesive and intentional.

---

# UX Philosophy

The UX should prioritize:

- clarity
- atmosphere
- hierarchy
- readability
- elegance
- compositional balance

The interface should feel:

- calm
- deliberate
- polished
- premium

Avoid:

- noisy UI
- cluttered layouts
- aggressive animations
- excessive interactions
- generic ecommerce widgets
- visual overload

The experience should guide attention naturally through composition and typography.

---

# Ecommerce Philosophy

This is not a marketplace-style store.

The shopping experience should feel:

- curated
- premium
- editorial
- product-focused

Product pages should:

- emphasize photography
- highlight craftsmanship
- communicate tone character
- feel immersive and premium

The UI should support product presentation rather than overpower it.

---

# Photography Philosophy

Photography is critical to the brand identity.

Images should feel:

- cinematic
- shadow-driven
- warm
- textured
- luxurious
- handcrafted

Use:

- dark backgrounds
- controlled reflections
- warm highlights
- selective lighting
- negative fill
- close-up detail compositions

Avoid:

- bright white ecommerce imagery
- flat lighting
- overexposed product photography
- sterile presentation

The photography should integrate naturally with the dark Art Deco visual system.

---

- product grid

- premium browsing experience

3. Single product page

---

# Technical Philosophy

The application should prioritize:

- maintainability
- explicit architecture
- SSR compatibility
- accessibility
- performance
- long-term clarity

The stack intentionally avoids:

- unnecessary abstractions
- excessive dependencies
- framework lock-in
- styling shortcuts

The project should remain:

- understandable
- composable
- scalable
- disciplined

---

# Design System Philosophy

The design system is a core architectural layer.

It should provide:

- visual consistency
- reusable primitives
- semantic APIs
- centralized styling rules
- predictable composition

The DS owns:

- typography
- spacing
- colors
- frames
- surfaces
- interactive styling
- layout primitives

Business/page components should compose DS components rather than implement styling directly.

---

# Long-Term Direction

The project should be structured so it can later support:

- expanded product catalogs
- more editorial content
- richer storytelling
- improved product photography
- enhanced SEO
- additional product categories
- future CMS integration if needed

without requiring architectural rewrites.

The MVP should establish a strong and coherent foundation.
